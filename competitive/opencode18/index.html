OPENCODE18 EXPERIENCE




What Is OpenCode18?

OpenCode18 is an online Open Source Competition organised by IIIT, Allahabad. This competition is coducted exactly one month before GSoC. As all open source competitions, even OpenCode first tells students and mentors to register and after a while, the projects are listed on the website.

This particular competition is really memorable for me because one of my projects got listed for this competition and I got selected as a mentor. My project Utlyz-CLI that I've been working on, for a while got selected, and obviously, I was extremely thrilled because now, my project was going to get more recognition. But I didn't leave it just there. I decided to take part in the competition as a participant too considering the fact that GSoC was nearing and I was still considerably new to Open Source. So I thought why not make the most out this oppourtunity!

In this blog post, I'm basically going to list out my contributions and experience as a participant and mentor. Hope you guys like it!

Experience As A Mentor

This is how my project looked before OpenCode18

My project, Utlyz-CLI got selected as a project to contribute to for all those participating in OpenCode18.

So what is this Utlyz-CLI and what does it do?
Well, it's basically a command line application that I made using Python. I always felt that it was difficult to navigate through to various tabs in the browser and search for what I want like, Facebook updates, searching google or wikipedia, getting news updates and so on and so forth. So I decided to make a command line application which gets everything you need from cricket scores to football transfer news, from facebook updates to google search, from news updates to song lyrics and what not just on typing a particular command. And BOOM, the result is right in front of you.

Now coming to my experience as a mentor!

My project didn't really look as professional as I would have wanted it to look. But after becoming a mentor, I got some magical powers in me. I felt like a leader now. I had to guide all those who were interested in contributing to my projects and it was tiresome sometimes. I must say, I had an amazing time mentoring people because it really helped me get a deeper knowledge on my project. Sometimes some guys would come up with some doubts, and I was mostly in a position to give them satisfactory answers. This gave me more confidence as a developer because I was able to assist people whenever they got stuck. It definitely helped me a lot as a developer because I got better at code reading too. Also, I've never been involved in a project such as this where about 5-6 people have contributed constantly. Although I don't know these guys personally, I'm really thankful to them for having contributed to my project. It helped me realise how important collaboration and team work is. Doing a project is one thing, and making your project public and asking others to contribute is another. This also gave me more confidence because there were 2 guys who contributed quite often and their contributions were somewhat major and the best part about this was that they actually understood all the existing codes. This made me happy because as a developer, you'd definitely want the entire coding fraternity to understand your code when they see it. So it felt really nice. Now we're done with this competition almost and I'm really happy with the outcomes from this.

  1. Now my project looks much more professional
  2. I've gained a lot of confidence now as a developer
  3. I learnt how to guide people on my own project
  4. I learnt how important outside contribution is
  5. I developed on my code reading skills

Experience As Participant

Contributions during OpenCode18

As a participant, I've contributed to 4 repositories namely PyDic, Grypto, OpenCode Collaborative and HackerSkills. Overall I have 17 PRs merged as you can see from the above image. I'll be talking in detail about all my contributions to various repositories and also I'll be telling about my experience as a participant.

My experience as a participant was perhaps the best so far. As a participant I wanted to learn a lot of new things and I'm sure I wouldn't have been able to learn those if I would've not participated in OpenCode18. I'd like to give credit the owners of the repositories for the same. So far, I've learnt many new things like Google's GoLang, how to use APIs, how to use the JSON Library, how to work with images, how to automate processes, how to read documentation of libraries properly and a lot more which I can't explain here because I'll just keep going on and on.

Some contributions that I'm proud of are converting text to speech(PyDic), creating a certificate generation program(HackerSkills), get crypto details for 4 different cryptos (Grypto)

I'm proud of converting text to speech because I've always only dreamt of it. Never thought it would actually be possible. But when I went deeper into it, I understood so much more and I realised I could achieve this. Obviously I did use an external library, but it still wasn't an easy task.
I'm proud of creating a certificate generation program because here I learnt how important and easy it is to automate such processes and eliminating man power.
The main reason why I'm proud of getting crypto details for 4 different cryptos is because I decided to contribute to this repo without even knowing what GoLang is. I had no clue about it. So over the course of the project I learnt Go Lang and contributed to that particular process. It felt amazing to do that.
